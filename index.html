Why Gutenberg?

  • For the first time ever, developers have the ability to completely revolutionize the WordPress editor using faster technologies like React and Node as compared to their legacy counterparts like jQuery.
  • With the Block API, extensions, integrations and customizations are now possible with a UI-first approach to development.
  • Gutenberg, will eventually take over theme development as a whole. Removing the need for the Customizer in areas like the header and footer and providing a truly “personalized” experience for users.
  • It’s the first peak into the future of WordPress

What you need to know

  • React: The Gutenberg Block API is written on top of the React Framework. An understanding of React Components, Life Cycles and concepts like props is detrimental for block development.
  • PHP: As blocks require PHP logic to be written in order to tie them to the database, the ability to “rewrite” your block on the server side is important. The API provides some nice ways to integrate, but make sure your PHP skills within the context of WordPress are sharp!
  • Javascript: A holistic understanding of Javascript is the backbone (no pun intended) of Gutenberg development. Make sure you understand technologies like Babel and Webpack if you plan on writing ES6. ES5 is of course supported.
  • CSS: New CSS layout modules like Flexbox and CSS Grid will open up a new world of CSS optimization and performance when it comes to Gutenberg.
